# 全球投资动能监控 PDF 产品规范

## 产品定位

`全球投资动能监控` 不是新闻摘要，而是把 X/KOL、微信公众号、微信群归档等信息转成可验证的投资动量报告。最终 PDF 面向两类读者：

- 专业投资者：快速看到信号强度、产业映射、风险触发和仓位态度。
- 普通投资者：用白话看懂市场在动什么、为什么动、该观察什么、什么情况下不要追。

## 固定目录

1. 一页决策总览
2. 今日核心判断
3. 主线深度拆解
4. 机会与仓位建议
5. 市场动量与分歧
6. 风险与证伪
7. 证据链
8. 明日行动清单

## 固定分析逻辑

每次报告按同一条逻辑链生成：

1. 最新信号：读取本次新增抓取内容。
2. 滚动上下文：读取最近几天 raw 归档，作为趋势基准。
3. 共识提炼：从少量高质量 KOL 中找最大公约数，而不是逐条复述。
4. 主线拆解：每条主线必须说明结论、支持论据、反向证据、国内映射和验证信号。
5. 交易价值：给出进攻、观察、等待确认或回避，并说明介入、加仓、减仓和证伪条件。
6. 风险证伪：说明判断错时会先出现什么反向信号。
7. 次日验证：列出下一天必须观察的指标、来源和行动规则。

## LLM 上下文控制

LLM 不直接读取全部原文。代码先做信息压缩：

- 每个来源至少保留一条高权重证据，避免单一大号淹没其他来源。
- 最新内容优先于历史上下文。
- 微信群归档会先过滤服务指南、闲聊和非投资噪音。
- 单条文本会截断，只保留可判断主题、逻辑和证据的摘要。
- 监控源少于 20 个时，首轮最多保留 18 条高权重记录，尽量覆盖不同博主/来源。
- 如果 LLM 仍失败，规则引擎生成同目录的兜底报告，保证 PDF 不空转。

## 两阶段 LLM 分工

LLM 被拆成两层，避免慢模型长时间承担成文任务：

1. Planner：默认 `kimi-k2.7-code`，只输出目录、核心判断、动量方向、证据强度和逻辑骨架。
2. Writer：默认 `minimax-m3`，只基于 Planner JSON 成文，不新增未经提纲支持的主题。
3. Fallback：Planner 默认回退 `minimax-m3`；Writer 默认回退 `doubao-seed-2.0-lite,deepseek-v4-flash`。
4. Schema gate：模型即使返回合法 JSON，也必须包含完整报告字段，否则视为失败并切换候选模型或规则兜底。

## 结构化中间层

LLM 输出 JSON，不直接输出最终 Markdown。代码负责把 JSON 渲染成固定报告模板，避免每天版式漂移。

核心字段：

- `one_page`：市场温度、风险等级、总体仓位、今日结论、普通投资者提示、今日不要做。
- `key_calls`：最重要的 2-4 条投资判断，包含证据等级、理由、国内映射和行动。
- `deep_dives`：主线深度拆解，包含核心论点、支持论据、反向证据、国内映射、操作含义和验证信号。
- `action_plan`：机会与仓位建议，包含介入、加仓、减仓/回避和证伪条件。
- `momentum_table`：主题动量变化和下一步观察。
- `source_view`：来源/博主分布，说明每个来源贡献了什么证据。
- `risk_radar`：风险触发、影响和应对。
- `evidence`：推文/文章/群消息摘要、链接或本地归档，以及支撑哪个结论。
- `tomorrow`：下一交易日验证清单和行动规则。

## PDF 输出原则

- Markdown 是归档格式，PDF 是阅读产品。
- PDF 渲染由代码和 CSS 控制，不依赖 LLM 自由排版。
- 表格用于承载决策信息，短句用于降低阅读负担。
- 证据链只保留摘要与链接，不附大段原文。
- 报告必须能在 Discord 中直接打开阅读。

## 后续可升级项

- 为主题热度、证据强度、拥挤度和风险等级增加可解释评分。
- 在 PDF 中加入热度条、风险色块和机会-风险象限图。
- 为普通投资者增加“不要做什么”的明确提示。
- 为专业投资者增加“观察池/交易池/剔除池”分层。
